HTML Tags and attributes

HTML uses tags to create elements in the website, and attributes and using within the tags to expand the information or style the elements. Below you will be able to find some of the most useful tags and related attributes used in HTML.

Structure and content

Let's begin with the structure of the website! Firstly, you have to declare your file with the !DOCTYPE html to signal the browser that the code is for HTML and to use the latest edition. Then you use HTML tag to start coding! The HEAD tag consists of all the information about the website, such as the title that will appear on the tab at the top of the site (with the tag TITLE), the LINK tags to link style sheets or fontfaces for styling via CSS, and any other important information. A very fast way of starting your document is to copy and paste the HTML 5 boilerplate, which will have most of the tags needed for the header already writen.

Following the HEAD tag, the BODY is where all the content of the website goes. We will go into more details soon.

Elements

Elements present in a website will always be within the BODY tag. If you want to have a title for a section, you use the h1 - h6 tags (bigger to smaller for semantic and flow reasons). If you want a paragraph to detail a section, you use the P tag (one is being used riight now!). To add an image you use the IMG tag, and the same applies for videos (VIDEO) and audio (AUDIO). These last ones are tricky because they dont behave like the rest of the tags. IMG tags are self-closing, so you only need one, and FIGCAPTION can be used to include some descriptive text that will stick with the image wherever it goes. VIDEO and AUDIO both require the SOURCE tag to implement the media we want. To add lists, both ordered (numeric list) or unordered (bullet points), you use the OL or UL respectively, and wrap every item of the list with the LIst tag (IMPORTANT, the only child of the OL or UL tags is LI. To add an anchor to a list, you have to wrap the anchor item with the list tag, not the other way round). To make a specific part of the text different to the rest, you can use SPAN, and style it in CSS. To add a button, you use the BUTTON tag, and you can style it however you need in CSS. Finally (for now), the TABLE tags are used to create tables within your website, and inside your table, you can have a rows, data, table headers, body and foot (TR, TD, THEAD, TBODY, and TFOOT respectively).

Semantics

For the sake of flow and comprenhension, we need semantics in our website and its code. Similar to how we have the head and body of the HTML code, within the body, we have a HEADER to include any relevant introductory information and titles, as well as the possibility of integrating a NAVigation list, the MAIN part of the body where the mayority of the content goes, and the FOOTER with all the remaining info about contact and copyright. Within the main, we have SECTIONs (just like this one, on semantics!). Information can also be split into DIVisions and ARTICLEs (you can check more about them in W3 school or MDN). Finally, to add something outside of the articles or sections, such as extra information not essential to the topic, you can use the ASIDE tag.

Attributes

The attributes is where it starts getting interesting. All tags can have attibutes, and these makes the document easier to read, the website better to style, and the elements to function. For the sake of brevity, we will make a table with all the information:

Attribute Description
Anchor tags I'm including them here because they control all the hyperlinks and whatnot
HREF (Hyperlink reference) Used to link a URL to anything with an anchor tag
SRC (source) Used to source an image, video or audio link. These last need the SRC to be inside the SOURCE tag. When adding a source, you can add a relative URL (local file) or an absolute URL (out sourced)
ALTernative Used in images, videos and audios to give a description when not rendered. Good practice for accessibility reasons.
WIDTH and HEIGHT Controls such attibutes of images
TITLE Gives more information about an element, but doesn't change it (unless styled in CSS). Furthermore, when hovering over a text that has the title attribute, more information will pop up.
STYLE Adds style directly into the element instead of using a CSS stylesheet. This is common mispractice as code ans style should be separate.

CSS and Styling your website

CSS, or Cascading Style sheet, is a document that goes linked to the HTMl page to give it some colour and style, along with any other changes you want to commit regarding display, location, fonts and all. As seen before, for readability reasons, the style is usually placed in a separate document. In CSS you use CSS declaration blocks, where you select and element, id, or class (to mention some), and assign a value to a property (e.g., h1 {color: blue;}, .class {color: blue;}, #id {color: blue;}...).

Color pallete and background

Colors can be split into foreground and background. The foreground is the colour of the letters, while the background colors is self-explanatory, and you can declare them with color and color-background respectively. Colors can be writen in 4 different ways:

  • Color word: CSS has around 150 different color words to choose from. Just write the color you want and, if it is in CSS, it will assign it.
  • Hexadecimal: You can use the format #XXXXXX, where the first, second and third pair of X will be a hexadecimal value to represent a hue of red, green and blue respectively. Whenever both X in a pair are the same number of letter, you ca choose to only write one (e.g., FF 00 00 can be writen as #F00).
  • RGB (Red green blue): when choosing a color with RGB, you must write RGB (x, y, z), where x, y, and z is a number from 0 to 255.
  • HSL (Hue saturation lightness): when choosing a color this way, you must write HSL (x, y, z) where x is a number from 0 to 100, and y and z are percentages.
  • Both RGB and HSL have an alternative mode where you add an A to the end of each, called an alpha value (RGBA and HSLA). This value, from 0 to 1, sets the transparency of the element. This way, instrad of making the whole element (letters and background) fully opaque or transparent, you can choose to only make the letters, or the background.

Within the color section, we will also mention that you can add an image to the background, and modify it in accordance to your needs (opacity, hue, size and location...). To do this, you must use the BACKGROUND-IMAGE, and use a URL to link the picture (can use either absolute or relative pathing).

Fonts and typography

The words in your website can be altered in many ways! You can change the size with FONT-SIZE, the weight with FONT-WEIGHT (100-900 to make it more or less bold), the color and the background (as we have seen before), the font with FONT-FAMILY (using both preloaded or linking external fonts using CSS and Fontfaces), the alignment with TEXT-ALIGN and the DISPLAY for more or less occupied space, and the style with FONT-STYLE.

Alignment, display and location

The alignment of the text can be styled like we have seen before. The display of text and images can be styled to ocupy a block, be inline, or both. Finally, the position can be left STATIC, to move it RELATIVE to its default position, to be ABSOLUTE (positioned relative to the first positioned ancestor element), FIXED somewhere in the site and not follow the flow of the document, or STICKY to become fixed after a certain part of the site is scrolled. To learn more, please check the MDN reference.

Pseudoclasses

When hovering over a button or a link, you can make the cursor change appearance using the CURSOR declaration. Furthermore, the links and text to be clicked can change color. This is done by setting a COLOR to the text by default, and then adding another declaration with the :HOVER pseudo-class to assign a color or style difference when hovering over. IMPORTANT - You must declare the change of cursor in all anchor links, for example, and not within the a:hover, as this might have unwanted changes.

When we want to change the color of links after visiting them, we use the :VISITED declaration. Finally, to customise the text when clicked but before it redirects you, you can use :ACTIVE. The Active declaration is important for buttons (standard, flat, skeuomorphic design...). When using Pseudoclasses, you must follow the :LINK, :VISITED, :HOVER, :ACTIVE cascade order so that elements show their states correctly (unvisited, visited, hovered and clicked).
